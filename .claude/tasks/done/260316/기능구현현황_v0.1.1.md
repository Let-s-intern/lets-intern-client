## 1. 프로그램 일정 화면 (`/mentor`)

멘토가 로그인 후 가장 먼저 보는 메인 화면입니다.
이번 주에 어떤 챌린지 피드백을 해야 하는지 캘린더 형태로 보여줍니다.

| 기능 ID | 기능 이름 | 구현 상태 | 설명 |
|---------|-----------|-----------|------|
| WK-01 | 주간 캘린더 | ✅ 완료 | 월~일 7일 캘린더에 챌린지 피드백 기간을 막대 바로 표시 |
| WK-02 | 주 이동 버튼 | ✅ 완료 | ← 이전주 / 다음주 → 버튼으로 주 이동 가능 |
| WK-03 | 주간 요약 카드 | ✅ 완료 | "이번주 전체", "오늘 마감", "미완료" 3개 카드 표시. 기존 API 데이터를 프론트에서 집계하여 표시 |
| WK-04 | 챌린지 필터 | ✅ 완료 | "전체" 또는 특정 챌린지만 골라서 캘린더에 표시 가능 |
| WK-05 | 기간 바 상태 숫자 | ✅ 완료 | 각 막대 바에 미제출/제출/시작전/진행중/완료 건수 표시 |
| WK-06 | 바 클릭 → 모달 | ✅ 완료 | 캘린더의 막대 바를 클릭하면 피드백 모달이 열림 |
| WK-07 | 환영 메시지 | ✅ 완료 | "OO 멘토님, 렛츠커리어와 함께..." 메시지 표시. 멘토 이름은 사용자 정보에서 자동으로 가져옴 |

## 2. 서면 피드백 모달

캘린더에서 막대 바를 클릭하면 열리는 팝업 창입니다.
멘토가 멘티의 미션 제출물을 확인하고 피드백을 작성하는 공간입니다.

| 기능 ID | 기능 이름 | 구현 상태 | 설명 |
|---------|-----------|-----------|------|
| FB-01 | 멘티 리스트 | ✅ 완료 | 왼쪽에 멘티 목록 표시, 각 멘티 옆에 상태 뱃지(시작전/진행중/완료) 색상 표시 |
| FB-02 | 멘티 정보 | ⚠️ 부분 구현 | 멘티 이름, 희망 직군, 희망 기업은 표시됨. **노션 링크는 하단 액션 버튼에만 있고 멘티 정보 영역에는 미표시** |
| FB-03 | 피드백 에디터 | ⚠️ 부분 구현 | 텍스트 입력은 가능하나 **단순 텍스트 입력창(textarea)으로만 구현됨**. 마크다운 서식(굵게, 목록 등)은 지원 안 됨. 향후 Lexical 에디터로 업그레이드 예정 |
| FB-04 | 임시 저장 | ✅ 완료 | "임시 저장" 버튼 클릭 시 서버에 피드백 저장, 상태가 "진행 중"으로 변경 |
| FB-05 | 최종 제출 | ✅ 완료 | "최종 제출" 버튼 클릭 시 확인 창 → 서버에 저장, 상태가 "완료"로 변경되며 수정 불가 |
| FB-06 | 상태 표시 바 | ✅ 완료 | "시작 전 → 진행 중 → 완료" 3단계 진행 상태를 시각적으로 표시 |
| FB-07 | 닫기 / 멘티 전환 | ✅ 완료 | 저장하지 않은 내용이 있으면 "저장하지 않고 이동하시겠습니까?" 확인 창 표시 |

## 3. 멘토 프로필 (`/mentor/profile`)

멘토의 기본 정보와 경력을 관리하는 페이지입니다.

| 기능 ID | 기능 이름 | 구현 상태 | 설명 |
|---------|-----------|-----------|------|
| PF-01 | 기본 정보 | ⚠️ 부분 구현 | 이름, 활동명, 전화번호, SNS, 이메일 입력 폼 있음. 이름/이메일/전화번호는 기존 사용자 정보에서 자동으로 채워짐. **프로필 이미지는 회색 원 자리만 있고 업로드 기능 없음** |
| PF-02 | 한줄 소개 | ✅ 완료 | 텍스트 입력 가능 (화면에서 입력만 되고 서버 저장은 PF-04에 의존) |
| PF-03 | 경력 관리 | ✅ 완료 | "경력 추가하기 +" 버튼으로 경력 카드 추가/삭제 가능. 회사명, 업무분야, 직책, 부서명, 재직기간 입력 가능 |
| PF-04 | 저장하기 | ❌ 미구현 | **저장 버튼은 있지만, 클릭하면 "프로필 저장 기능은 준비 중입니다" 알림만 표시**됨. 백엔드에 프로필 저장 API가 만들어져야 연결 가능 |

## 4. 인프라 (기반 구조)

멘토 페이지가 동작하기 위한 기본 골격입니다.

| 구성 요소 | 구현 상태 | 설명 |
|-----------|-----------|------|
| 멘토 라우트 | ✅ 완료 | `/mentor`, `/mentor/profile` 경로로 독립 페이지 생성됨 |
| 멘토 접근 제어 (Guard) | ✅ 완료 | 멘토가 아닌 사용자가 접속하면 메인 페이지로 이동시킴. **현재 테스트를 위해 임시 비활성화 상태** |
| 사이드바 | ✅ 완료 | 왼쪽에 "프로그램 일정", "Profile" 메뉴 표시 |
| Provider | ✅ 완료 | 멘토 전용 데이터 캐시 관리 (admin/user와 분리) |

## 5. 서버 연동 상태

### 연결 완료된 API (데이터가 실제로 오가는 것)

| 기능 | API | 상태 |
|------|-----|------|
| 멘토 챌린지 목록 조회 | `GET /api/v1/challenge-mentor` | ✅ 연결됨 |
| 미션별 피드백 목록 조회 | `GET /api/v1/challenge/{id}/mission/feedback` | ✅ 연결됨 |
| 멘티 제출 목록 조회 | `GET /api/v1/challenge/{id}/mission/{id}/feedback/attendances` | ✅ 연결됨 |
| 멘티 피드백 상세 조회 | `GET /api/v1/challenge/{id}/mission/{id}/feedback/attendances/{id}` | ✅ 연결됨 |
| 피드백 저장/제출 | `PATCH /api/v1/attendance/{id}/mentor` | ✅ 연결됨 |
| 멘토 여부 확인 | `GET /api/v2/user/is-mentor` | ✅ 연결됨 |
| 사용자 정보 조회 | `GET /api/v1/user` | ✅ 연결됨 |

### 백엔드 API가 필요한 기능 (아직 서버에 API가 없음)

| 기능 | 필요한 API | 현재 처리 방식 |
|------|-----------|---------------|
| 프로필 저장 | `PATCH /v2/mentor/profile` | "준비 중" 알림 표시 |
| 프로필 조회 | `GET /v2/mentor/profile` | 기존 사용자 정보(이름/이메일/전화번호)만 채움 |
| 프로필 이미지 업로드 | S3 업로드 API | 회색 자리 표시만 있음 |

## 6. 알려진 제한사항

| 항목 | 내용 | 영향 |
|------|------|------|
| 멘토 계정 필요 | 현재 테스트 계정이 멘토로 등록되지 않아 `/mentor` 접속 시 메인으로 이동됨 | Guard를 임시 비활성화하여 우회 중 |
| 단순 텍스트 에디터 | 피드백 작성 시 마크다운 서식 사용 불가 | 글자 입력/저장은 정상 작동 |
| 프로필 저장 불가 | 입력한 프로필 정보가 서버에 저장되지 않음 | 페이지 새로고침 시 입력 내용 사라짐 |
| 프로필 이미지 | 업로드 기능 없음 | 기본 아이콘만 표시 |
| 모바일 미대응 | 모바일 화면 레이아웃 미최적화 | PC에서만 정상 표시 |
