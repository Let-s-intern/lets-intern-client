## 진행 보고서

### 완료일: 2026-02-24

### 1.1 — comparisons.ts label 불일치 수정

**원인 분석**: `FREQUENT_COMPARISON[0].right`의 값이 `'대기업 자기소개서 완성 챌린지'`(공백 포함)였으나, `CHALLENGE_COMPARISON[4].label`은 `'대기업 자기소개서완성 챌린지'`(공백 없음)로 달랐음. `findProgramIdByLabel()`이 exact match를 사용하므로 null을 반환, 클릭 시 아무 동작도 하지 않았음.

**수정 내용**: `FREQUENT_COMPARISON[0].right` 값에서 공백 1칸 제거.

---

### 1.2 — RecommendedComparisons 정렬 수정

**원인 분석**: `ChallengeCompareSection.tsx`의 `w-[73.75rem]` 컨테이너 내에서 `RecommendedComparisons`는 좌측 정렬, 카드들은 `justify-center`로 중앙 정렬되어 있어 시각적 불일치 발생.

**수정 내용**: `RecommendedComparisons` 렌더링 위치를 `<div className="flex justify-center">` 래퍼로 감싸 카드 그리드 중심과 정렬 통일.

---

### 2.1 — CompareResultCard CTA 버튼 정렬 수정

**원인 분석**: `CompareResultCard.tsx`의 CTA 버튼 컨테이너가 `flex justify-end gap-[50px] px-2.5`으로 설정되어 버튼들이 카드 우측 끝으로 밀렸음. 썸네일 헤더는 `pl-[120px]` 기준인데 버튼은 다른 정렬 방식을 사용.

**수정 내용**: `flex justify-end gap-[50px] px-2.5` → `flex gap-10 pl-[7.5rem]`으로 변경하여 썸네일과 동일한 기준점으로 정렬.

---

### 3.1 — ResultSection width 반응형 수정

**원인 분석**: 결과가 1개일 때 `w-[600px]`로 고정되어 있어 2개일 때와 동일한 너비처럼 보이고, 뷰포트가 좁을 때 넘침.

**수정 내용**: `w-[600px]` → `w-full max-w-[37.5rem]`으로 변경. 뷰포트 너비에 따라 유연하게 조절되면서 최대 37.5rem(600px) 제한 유지.

---

### 3.2 — CurationScreen 좌우 여백 반응형 수정

**원인 분석**: `CurationScreen.tsx`에서 `px-[120px]`(7.5rem), `mx-[120px]`(7.5rem) 고정값 사용으로 좁은 뷰포트에서 컴포넌트들이 찌그러짐.

**수정 내용**:
- 폼 섹션: `px-[120px]` → `px-6 md:px-10 lg:px-[7.5rem]`
- FAQ 섹션: `px-[120px]` → `px-6 md:px-10 lg:px-[7.5rem]`
- 섹션 divider(2곳): `mx-[120px]` → `mx-6 md:mx-10 lg:mx-[7.5rem]`

---

### 3.3 — ChallengeCompareSection 고정 너비 반응형 수정

**원인 분석**: `w-[1180px]`(73.75rem), `w-[1000px]`(62.5rem) 고정으로 뷰포트보다 넓어지면 가로 스크롤 발생.

**수정 내용**:
- 메인 컨텐츠 컨테이너: `w-[1180px]` → `w-full max-w-[73.75rem]`, 소형 화면용 `px-6 md:px-10 lg:px-0` 추가
- 섹션 헤더 내부: `w-[1000px]` → `w-full max-w-[62.5rem]`

### 4.1 — 카드 고정 너비 반응형 처리

**원인 분석**: Row1(4장), Row2(3장) 고정 분리 방식과 `w-[240px]` 카드 고정 너비로 좁은 화면에서 넘침.

**수정 내용**:
- Row1/Row2 분리 → 단일 `flex flex-wrap justify-center gap-6 py-5` 컨테이너로 통합
- 카드 너비: `w-[240px]` → `w-[15rem]` (rem 통일), 썸네일 `w-[240px]` → `w-full` (카드 너비에 맞게 유동)

---

### 4.2 — CompareResultCard 내부 고정 너비 반응형 처리

**원인 분석**: `w-[300px]` 고정값이 썸네일 열, InfoRow 값 열, CTA 버튼에 사용되어 좁은 뷰포트에서 넘침.

**수정 내용**:
- 썸네일 컨테이너: `w-[300px]` → `flex-1`, 이미지: `w-[200px]` → `w-full max-w-[12.5rem]`
- InfoRow 값 열: `w-[300px]` → `flex-1 min-w-0`
- CTA 버튼: `w-[300px]` → `flex-1`, `h-[46px]` → `h-[2.875rem]` (rem 통일)
