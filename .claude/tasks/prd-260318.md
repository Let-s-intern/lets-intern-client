> 작성일: 2026-03-18
> 브랜치: LC-2909-렛츠커리어-챌린지-피드백-멘토링-상세-안내-페이지

---

## 1. 개요

챌린지 상세페이지에서 연결되는 **피드백 멘토링 옵션 안내 랜딩페이지**를 제작한다.
서버/어드민 연동 없이, JSON 기반 정적 데이터로 관리하며 프리미엄 느낌의 랜딩페이지를 구현한다.

---

## 2. 페이지 URL 및 라우팅

- **경로**: `/challenge/feedback-mentoring` (쿼리 파라미터로 챌린지 구분)
- **URL 예시**: `/challenge/feedback-mentoring?challenge=marketing`
- **URL의 challenge 파라미터**에 따라 해당 챌린지 메뉴가 기본 선택되고, 관련 콘텐츠 표시
- 파라미터 없을 시 첫 번째 챌린지가 기본 선택

---

## 4. 섹션 상세 설계

### 4.1 HeroSection (01_상단)

- **배경**: 어두운 남색/보라색 그라데이션
- **텍스트**:
  - 소제목: "렛츠커리어 현직자 멘토단과 함께하는"
  - 메인 카피: "혼자 준비하는 취업은 이제 그만, **현직자와 함께**"
  - 서브 카피: "대기업 · IT 대기업 · 시리즈 B 이상 스타트업 현직자들이 **여러분의 서류를 직접 진단합니다**"
- **애니메이션**: 별(✦)들이 반짝거리는 CSS 애니메이션 (배경 곳곳에 배치)
- **장식 요소**: 상단 좌우에 서류 아이콘 이미지 (기울어진 형태)

### 4.2 ChallengeMenuSection (02_챌린지 메뉴)

- **타이틀**: "어떤 챌린지의 피드백이 궁금하신가요?"
- **메뉴 목록** (2줄 배치):
  - 1줄: 기필코 경험정리 챌린지 | 이력서 1주 완성 챌린지 | 자기소개서 2주 완성 챌린지
  - 2줄: 포트폴리오 2주 완성 챌린지 | 대기업 완성 챌린지 | 마케팅 서류 완성 챌린지 | HR 서류 완성 챌린지
- **인터랙션**:
  - 호버 시 텍스트 색상/밑줄 변경
  - 클릭 시 활성 상태 표시 (보라색 하이라이트)
  - 클릭한 챌린지에 따라 03~06 섹션 콘텐츠 변경
  - URL 쿼리 파라미터 동기화
- **동작**: 07~09 섹션은 공통이므로 변경하지 않음

### 4.3 FeedbackIntroSection (03_챌린지 피드백 소개)

- **좌측**: 피드백 리포트 이미지 (챌린지별 상이)
  - 타이틀: "[챌린지명] 서류 피드백 REPORT"
  - 배지/카피: "3,000명의 데이터를 보유한", "취업 연구팀, 현직자 피드백으로", "서류 합격률 200% 고공 상승!"
- **우측**: 피드백 옵션 카드
  - **STANDARD**: 피드백 횟수, 피드백 개수, 진행 방식, 멘토 정보
  - **PREMIUM**: 피드백 횟수, 피드백 개수, 진행 방식, 멘토 정보
  - "피드백 자세히 보기 →" 클릭 시 서면 피드백 이미지 표시
- **조건부 표시**: 옵션이 있는 챌린지만 표시

### 4.4 MentorListSection (04_챌린지 멘토 소개)

- **타이틀**: "[챌린지 분야명], **현직자 N명**이 다 봐드립니다" (챌린지별 상이)
- **카드 리스트**: 멘토 카드 가로 나열
  - 회사 로고
  - 회사명
  - 직무
  - 멘토 실루엣/사진
- **데이터**: 챌린지별 다른 멘토 목록

### 4.5 BeforeAfterSection (05_챌린지 비포 에프터)

- **조건부 표시**: 자료가 있는 챌린지만 표시
- **레이아웃**: Before(좌) / After(우) 비교
  - Before: 평범한 포트폴리오 캡처
  - After: 핵심 역량이 돋보이는 포트폴리오
- **설명 텍스트**: 각 이미지 하단에 설명 카피

### 4.6 LiveMentoringSection (06_라이브 멘토링)

- **조건부 표시**: 라이브 멘토링이 있는 챌린지만 표시
- **타이틀**: "1:1 LIVE 피드백, 영상으로 미리 확인하세요!"
- **서브카피1**: "혼자 막막했던 고민들, 멘토님과 실시간으로 해결하세요"
- **영상**: 유튜브/영상 임베딩 (재생 가능)
- **서브카피2** (영상 하단): "라이브로 주고 받는 맞춤형 피드백으로, 서류 완성도 UP!" (피드백 반영)
  - 글씨 크기는 서브카피1과 동일

### 4.7 UserReviewSection (07_유저 후기) - 공통

- **타이틀**: "수강생의 피드백 솔직 후기"
- **후기 카드**: 캡처 이미지 기반 후기
  - 전면 2~3개 카드 선명하게
  - 뒤쪽 카드들은 흐릿하게 처리 (blur 효과)하여 후기가 많아 보이도록 연출
- **레이아웃**: 카드 약간 기울여서 캐주얼하게 배치

### 4.8 SuccessStoriesSection (08_취업에 성공한 사람들) - 공통

- **타이틀**: "렛츠커리어와 함께 취뽀한 주인공들을 소개합니다"
- **Swiper 롤링 카드**: 기존 메인 페이지의 성공 사례 컴포넌트 재사용/참고
  - 클래스: `swiper swiper-initialized swiper-horizontal slide-per-auto slide-rolling`
  - 배경 없이 카드만 롤링
- **카드 정보**: 회사명, 직무, 이름(가명), 합격 연도, 회사 로고

### 4.9 ApplyCtaSection (09_챌린지 신청하기)

- **버튼**: "[챌린지명] 신청하러 가기"
- **동작**: 선택된 챌린지의 상세 페이지로 이동
- **URL 연동**: 챌린지 메뉴 선택에 따라 버튼 텍스트와 이동 경로 동적 변경

## 5. 데이터 관리

서버/어드민 연동 없이 **JSON/TypeScript 정적 데이터**로 관리.
데이터 원본: `.claude/tasks/data.md`
에셋 원본: `.claude/tasks/asset/` (개발 시 `public/` 등으로 복사 필요)

### 5.1 챌린지별 피드백 데이터 요약

| 챌린지 | BASIC | STANDARD | PREMIUM | 피드백 방식 | 라이브멘토링 |

### 5.2 멘토 데이터 관리

멘토 정보는 데이터 파일 하나에서 전부 관리할 수 있어야 한다:
- **멘토별 설정 가능 항목**: 닉네임, 회사명, 직무, 프로필 이미지 경로
- **챌린지별 설정 가능 항목**: 표시할 멘토 수(displayCount), 멘토 목록
- **프로필 이미지**: 개별 멘토마다 다른 이미지 지정 가능. 현재 4장의 프로필 이미지를 돌려 사용
- **추후 확장**: 이미지/멘토 정보만 데이터 파일에서 수정하면 바로 반영

현재 사용 가능한 프로필 이미지 (4장, 돌려쓰기):

- `/images/challenge-feedback/mentors/mentor-2.png`
- `/images/challenge-feedback/mentors/mentor-3.png`

## 6. 디자인 가이드

### 6.1 전체 톤앤매너
- **프리미엄 랜딩페이지** 느낌
- 어두운 남색/보라색 배경 (다크 테마)
- 포인트 컬러: 보라색(#7C6BFF 계열), 흰색 텍스트
- 일부 섹션(비포에프터)은 밝은 배경 가능

### 6.2 타이포그래피
- 메인 타이틀: bold, 큰 사이즈
- 서브 타이틀: medium/regular, 중간 사이즈
- 본문: regular, 표준 사이즈
- **일관성**: 섹션 간 폰트 크기/간격 통일

### 6.3 간격 규칙
- 섹션 간 간격: 통일 (py-20 ~ py-24 정도)
- 좌우 마진: 통일 (max-w-[1200px] mx-auto px-6 등)
- 카드 간 간격: gap-6 또는 gap-8

### 6.4 애니메이션/인터랙션
- 별 반짝임: CSS keyframes (opacity, scale 변화)
- 챌린지 메뉴 호버: 텍스트 색상 변경 + 밑줄
- 챌린지 메뉴 클릭: 활성 보라색 하이라이트
- 섹션 진입 시: fade-in 또는 slide-up (세련된 효과)
- Swiper 롤링: 자동 무한 스크롤

## 8. 주의사항

1. **공통 컴포넌트** 재사용 가능한 것은 활용 (Button, Layout 등)
2. **서버 연동 없음** — JSON/TS 파일로 데이터 관리
3. **섹션 교체 가능** — 챌린지마다 필요한 섹션이 다르므로 조건부 렌더링
4. **폰트 일관성** — 섹션 간 타이포그래피 통일
5. **섹션 간격 통일** — padding/margin 일관적 적용
6. **좌우 마진 통일** — 컨테이너 max-width + padding 통일
