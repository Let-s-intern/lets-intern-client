## 주의할 점

1. 맨트들 폰트를 보면 일관성이 없을 수 있어 어느정도 일관성 있게 작성해야해
2. 각 섹션들간의 간격을 통일성 있게 구성해
3. 좌우마진 통일감 줘
4. 에니메이션과 효과를 적절하게 세련되게 넣어줘
5. 첼린지에는 옵션이 있어 베이직 부터 프리미엄?까지 있는걸로알아 그런데 첼린지 상세페이지에는 첼린지에 대한 홍보내용만 있어서 부가적으로 설명해줄 페이지를 만들고 있어
   1. 첼린지옵션은 가격대가 있는 편이고 옵션을 붙인다는거기 때문에 어느정도 프리미엄 느낌으로 만들거야
   2. 상세페이지에서 연결될 페이지라서 정보중심의 qna보다 랜딩페이지 느낌의 상세페이지를 만들고자해

## 개발 주의사항

1. 공통 컴포넌트 ui를 사용할 수 있는게 있다면 사용할 것
2. 지금 이 페이지는 어드민이나 서버에 연동시키지 않고 제작될거야 설정과 내부 요소 맨트들을 쉽게 설정할 수 있게 json이나 다른 방식으로 데이터를 쉽게 관리할 수 있게 작성해
3. 각 섹션이 항상 있어야 하는게 아니라 첼린지 마다 필요한 섹션이 달라질 수 있음을 유의하고 섹션을 쉽게 교체할 수 있게 해줘
4. 현제 이 페이지 구현을 폴더 어디에 배치해야할지 아직 몰라 적절한 페이지 위치를 선정해야해
5. task를 만들때 잘모르거나 해깔리는 부분은 바로바로 나한테 다시 질문을 시작해
6. task가 진행될때는 task과정이 끝나기 전까지 나한테 다시 질문하지 말라고 기획서에 작성해줘

task 작성법을 알려줄게
각 task는 푸시 단위로 작성하고 세부 task는 커밋 단위로 작성해
하나의 테스크가 끝나면 테스트 코드를 간략하게 작성하고 작성한 파일에 대한 린팅 및 포메팅을 진행해
오류가 발생하면 세부 task를 하나 더 만들고 오류를 수정하고 반복해서 개발을 진행해

.claude/skills/task-runner/SKILL.md task파일에 테스크 러너를 사용해서 개발하고
개발 진행시에는 .claude/roles/developer.md 이 에이전트를 이용해서 개발해
필요에 따라서 팀에이전트를 사용해서 개발해 에이전트들의 룰은 .claude/roles 여기에 저장되어 있어
task에 docs폴더에서 참고해야하는 문서가 있다면 task에 docs에 어떤 문서를 확인해야하는지 문서의 경로를 알려주는 코멘트도 작성해줘

각 테스크의 푸시단위가 끝난다면 클로드코드 컴팩트를 진행해
너무 많이 싸여서 멈추는걸 방지해줘
진행하면서 따로 기록해야할 문서가 생긴다면 docs나 task에 md파일로 적절하게 작성해서 추후 개발에 있어서 도움이 될 수 있도록 해
