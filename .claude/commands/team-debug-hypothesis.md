# Debug Hypothesis Team 진입점

당신은 [.claude/teams/debug-hypothesis.md](../teams/debug-hypothesis.md) 레시피에 따라 경쟁 가설 검증 팀을 운영합니다.

## 사용자 요청

$ARGUMENTS

## 실행 지침

1. **레시피 읽기**: [.claude/teams/debug-hypothesis.md](../teams/debug-hypothesis.md)를 먼저 읽습니다.

2. **가설 수립 (메인 세션에서 먼저)**:
   - 버그 증상을 정리하고 **3~5개 경쟁 가설** 작성
   - 각 가설은 서로 배타적이거나 검증 방법이 달라야 함

3. **공식 Agent Team 생성 권장** (`TeamCreate`):
   - **이유**: 워커끼리 서로의 가설을 반증해야 함 → 직접 메시지 통신 필요
   - 서브에이전트로는 디베이트 불가 (서브에이전트끼리 대화 X)
   - 가설 1개당 워커 1명 (총 3~5명, `task-executor` 또는 일반 워커)

4. **각 워커에게 전달**:
   - 본인이 검증할 가설
   - 다른 워커들의 가설 목록 (반증 시도용)
   - 재현 시나리오, 관련 로그/스택트레이스, 의심 코드 경로

5. **수렴 규칙**:
   - 워커들이 자신의 가설을 검증하면서 다른 가설을 반증
   - 살아남은 가설이 실제 원인일 확률이 높음
   - 리드는 결론을 사용자에게 보고하고 수정안 제시

6. **단순 디버깅이면 팀 X**: 재현 가능하고 가설이 1개 명확하면 단일 세션으로 처리.
