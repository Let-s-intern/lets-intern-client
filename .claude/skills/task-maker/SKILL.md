# PRD 기반 작업 목록 생성

PRD와 요구사항을 분석하여 Git 워크플로우에 맞는 구조화된 작업 목록을 생성합니다.

---

## 작업 목록 생성 프로세스

### Step 1: 입력 분석

- PRD 파일이 지정된 경우: 기능 요구사항, 사용자 스토리 및 모든 섹션을 분석
- PRD 없이 요청된 경우: 핵심 요구사항을 추출하고 합리적 가정을 설정 (가정은 명시)

### Step 2: Phase 1 — 상위 작업 생성

- PRD 분석 기반으로 **3~5개**의 상위 작업 생성
- 하위 작업 없이 상위 작업만 먼저 제시
- 사용자에게 전달:
  > "PRD를 기반으로 상위 레벨 작업을 생성했습니다. 하위 작업을 생성할 준비가 되었나요? 진행하려면 'Go'로 응답해주세요."
- 사용자가 **"Go"** 로 응답할 때까지 대기

### Step 3: Phase 2 — 하위 작업 생성

사용자 확인 후 각 상위 작업을 하위 작업으로 분해:

- 각 하위 작업은 **하나의 Git 커밋** 단위
- 커밋 하나로 처리하기 어려운 경우 → 하위의 하위 작업(3단계)으로 분해

**커밋 단위 분해 기준** (하나라도 해당 시):
- 파일 수정이 5개 이상
- 구현 예상 시간이 2시간 이상
- 독립적으로 테스트 가능한 여러 기능 포함
- 로직이 복잡하여 단계별 구현 필요

### Step 4: 테스트 작업 추가

모든 구현 하위 작업에 자동으로 테스트 하위 항목 생성:
- `[번호].1` 테스트 코드 작성
- `[번호].2` 테스트 실행 및 검증
- `[번호].3` 오류 수정 (필요 시)

### Step 5: 관련 파일 식별

- 생성/수정할 파일을 식별하여 `관련 파일` 섹션에 나열
- 각 파일에 목적 한 줄 설명 작성

### Step 6: 파일 저장

- **위치**: `.claude/tasks/`
- **파일명**: `tasks-[prd-파일명]-[푸시단위].md`
  - 예: PRD가 `prd-user-profile.md` → `.claude/tasks/tasks-prd-user-profile-1.md`

## 작업 계층 구조

| 레벨 | 예시 | Git 단위 | 기준 |
|---|---|---|---|
| 상위 작업 | 1.0, 2.0 | **Push** | 배포 가능한 기능 단위, 3~7개 하위 작업 |
| 하위 작업 | 1.1, 1.2 | **Commit** | 독립적으로 실행·검증 가능 |
| 하위의 하위 | 1.1.1, 2.1.1 | **Commit** | 하위 작업이 너무 클 때 분해 |

### 상위 작업 구성 기준 (Push 단위)

- **기능적 완결성**: 독립적인 기능 제공
- **배포 가능성**: Push 후 배포해도 문제없는 상태
- **적절한 범위**: 하위 작업 3~7개 권장

❌ 잘못된 예 (너무 큰 단위):
```
- [ ] 1.0 사용자 관리 시스템 구현
    - [ ] 1.1 등록 / 1.2 로그인 / 1.3 프로필 / 1.4 권한 / 1.5 검색 / 1.6 삭제
```

✅ 올바른 예:
```
- [ ] 1.0 사용자 인증 기능 구현
    - [ ] 1.1 등록 / 1.2 로그인 / 1.3 로그아웃
- [ ] 2.0 사용자 프로필 관리
    - [ ] 2.1 조회 / 2.2 수정 / 2.3 이미지 업로드
```

## 출력 형식

```markdown
### 관련 파일

- `path/to/component.tsx` - 주요 컴포넌트
- `path/to/api.ts` - API 핸들러

## 작업

- [ ] 1.0 상위 작업 제목 (Push 단위)
    - [ ] 1.1 하위 작업 설명 (커밋 단위)
        - [ ] 1.1.1 테스트 코드 작성
        - [ ] 1.1.2 테스트 실행 및 검증
        - [ ] 1.1.3 오류 수정 (필요 시)
    - [ ] 1.2 하위 작업 설명 (커밋 단위)
        - [ ] 1.2.1 테스트 코드 작성
        - [ ] 1.2.2 테스트 실행 및 검증
        - [ ] 1.2.3 오류 수정 (필요 시)

- [ ] 2.0 상위 작업 제목 (Push 단위)
    - [ ] 2.1 복잡한 하위 작업 (작업량이 많은 경우)
        - [ ] 2.1.1 세부 구현 작업 1 (커밋 단위)
            - [ ] 2.1.1.1 테스트 코드 작성
            - [ ] 2.1.1.2 테스트 실행 및 검증
            - [ ] 2.1.1.3 오류 수정 (필요 시)
        - [ ] 2.1.2 세부 구현 작업 2 (커밋 단위)
            - [ ] 2.1.2.1 테스트 코드 작성
            - [ ] 2.1.2.2 테스트 실행 및 검증
            - [ ] 2.1.2.3 오류 수정 (필요 시)
```

## 생성 완료 체크리스트

- [ ] PRD의 모든 기능 요구사항이 작업으로 변환됨
- [ ] 각 상위 작업이 Push 단위로 적절 (배포 가능한 단위)
- [ ] 각 하위 작업이 하나의 커밋 단위로 적절
- [ ] 복잡한 작업은 하위의 하위로 분해됨
- [ ] 모든 구현 작업에 테스트 작업 포함
- [ ] 관련 파일 목록이 정확하고 설명이 명확
- [ ] 작업 번호 체계가 일관됨
- [ ] 파일이 `.claude/tasks/` 디렉토리에 올바른 이름으로 저장됨
- [ ] 상위 작업 간 의존성이 최소화됨

---

## 행동 원칙

- **구체적으로**: 모호한 표현을 측정 가능한 결과로 대체
- **간결하게**: 불필요한 설명 배제
- **현실적으로**: 작업 규모를 실제 복잡도에 맞게 산정
- **확인 우선**: 입력이 모호하면 합리적 가정을 세우되, 가정을 명시
- **과잉 설계 금지**: 단순한 버그 수정에 8개 하위 작업을 만들지 않음
