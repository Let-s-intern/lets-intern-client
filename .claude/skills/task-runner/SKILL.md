# 작업 목록 실행 워크플로우

`.claude/tasks/` 디렉토리의 작업 목록 마크다운 파일을 읽고 순서대로 실행합니다.

---

## 시작 절차

1. 사용자가 지정한 작업 파일을 읽는다 (예: `.claude/tasks/tasks-prd-user-profile-1.md`)
2. 파일이 지정되지 않은 경우 → `.claude/tasks/` 디렉토리의 파일 목록을 보여주고 선택 요청
3. 첫 번째 미완료(`[ ]`) 하위 작업을 찾아 실행 시작

---

## 작업 진행 원칙

- **한 번에 하나의 상위 작업만 수행한다.**
- 상위 작업 완료 후 사용자에게 승인(yes/y)을 받기 전까지 다음 상위 작업을 시작하지 않는다.
- 하위 작업은 허락 없이 연속 진행한다.

## 완료 처리

1. 하위 작업 완료 → 즉시 `[ ]`를 `[x]`로 변경
2. 모든 하위 작업 `[x]` → 상위 작업도 `[x]`로 변경
3. 각 상위 작업 완료 후 → 사용자 승인 대기

## Git 커밋 규칙

### 커밋 시점

- **하위 작업 완료 시**: 즉시 커밋
- **상위 작업 완료 시**: Git push 수행

### 커밋 메시지 형식

```
type(task_num): Subject
body
footer
```

- **type**: `feat` | `fix` | `docs` | `style` | `refactor` | `test` | `chore`
- **Subject**: 최대 50자, 마침표 없이, 한국어 우선 (영문 시 동사 원형 시작, 첫 글자 대문자)
- **Body** (선택): 무엇을, 왜 했는지 작성. 최대 75자. 영어로 작성
- **Footer** (선택): `Fixes: #이슈` | `Resolves: #이슈` | `Ref: #이슈` | `Related to: #이슈`

예시:
```
fix(task 2.1.2): 로그인 기능 구현

Add JWT-based authentication flow

Resolves: #42
```

## 실행 지침

1. 다음 하위 작업이 무엇인지 확인
2. 작업 수행
3. 테스트 코드 작성 → 실행 → (오류 시) 오류 수정
4. 작업 목록 체크 (`[x]`)
5. Git 커밋
6. 사용자 승인 대기 → 승인 시 다음 작업 진행

### 오류 발생 시

- 해당 하위 작업 아래에 `1.1.3.1`, `1.1.3.2` 형식으로 세부 수정 작업 추가

### 관련 파일 관리

- 생성/수정된 모든 파일을 관련 파일 섹션에 나열
- 각 파일에 목적 한 줄 설명 작성
