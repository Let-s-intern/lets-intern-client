# 코드 품질 가이드

좋은 프론트엔드 코드 = **변경하기 쉬운 코드**.
새로운 요구사항을 구현할 때 기존 코드를 수정하고 배포하기 수월한 코드가 좋은 코드다.

4가지 기준으로 판단한다: 가독성, 예측 가능성, 응집도, 결합도.

## 빠른 판단 기준

코드를 작성하거나 리뷰할 때 아래 순서로 체크한다:

1. **읽기 쉬운가?** — 한 번에 고려하는 맥락이 적고, 위에서 아래로 자연스럽게 읽히는가
2. **예측 가능한가?** — 이름, 파라미터, 반환 값만 보고 동작을 알 수 있는가
3. **함께 수정되는 코드가 같이 있는가?** — 한 곳을 고치면 관련된 곳도 자연스럽게 고쳐지는가
4. **수정 영향 범위가 좁은가?** — 이 코드를 고쳤을 때 다른 곳이 깨지지 않는가
